# 🤖 자동 코치 에이전트 (Agent Coach)

실무형 규칙과 에이전틱(Agentic) 워크플로우를 통해 `Node.js` 및 `Docker` 환경의 과제를 자동으로 진단하는 시뮬레이션 프로젝트입니다.

---

## 🎯 1. 프로젝트 목표

- **에이전트 구조 이해**: LLM 없이도 [계획(Plan) → 실행(Execute) → 검증(Verify)]으로 이어지는 에이전트의 사고 흐름을 코드로 구현합니다.
- **상태 관리와 도구 분리**: 에이전트 내부 상태(State)와 외부에서 작동하는 도구(Tools)를 명확히 분리하여 구조적 설계를 익힙니다.
- **진단 프로세스의 시각화**: 에이전트가 어떤 근거로 판단을 내리는지 사용자에게 **Trace(추척 로그)**를 제공하여 신뢰성을 높입니다.

## 💡 2. 알아야 할 핵심 개념

1. **에이전틱 워크플로우**: 단순한 API 호출이 아니라, 상황에 따라 스스로 다음 행동을 결정하고 필요시 사용자에게 추가 정보를 요청하는 흐름입니다.
2. **규칙 기반 진단 (Rule-based)**: 실무에서 흔히 발생하는 실수 패턴(예: 컨테이너 내부 127.0.0.1 사용 등)을 정규표현식으로 정의하여 진단합니다.
3. **Trace (진단 로그)**: 사용자가 "왜 이런 결과가 나왔지?"라고 의심하지 않도록 에이전트의 내부 판단 과정(Stage별 메세지)을 노출합니다.

## 🔄 3. 에이전트 작동 흐름

1. **[PLAN]**: 사용자가 입력한 내용(에러 로그, 설정 파일)을 프리셋 규칙과 비교하여 분석 계획을 세웁니다.
2. **[EXECUTE]**:
   - **정적 분석**: 정규표현식 도구를 사용하여 코드 내 의심 항목과 누락된 정보를 찾습니다.
   - **동적 분석**: 실제 DB 환경이 준비된 경우 쿼리를 실행하여 연결 상태를 확인합니다.
3. **[VERIFY]**:
   - 정보가 충분하면 **DONE**(진단 완료 및 조치 가이드)을 반환합니다.
   - 필수 정보가 부족하면 **NEED_INFO**(추가 정보 요청)를 통해 사용자에게 힌트를 줍니다.
   - 위험한 명령어가 포함된 경우 **FAIL**(이상 감지)로 중단합니다.

## 📂 4. 실행 방법

```bash
docker compose up -d --build

docker ps

docker compose logs -f --tail=100 api
```
